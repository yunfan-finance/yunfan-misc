# GPU 算力经济学与 Oracle 财务深度拆解（第二版）

## 第一部分：GPU 的经济账

### 1.1 过去的黄金时代：A100 和 H100 的盈利现实

2020-2024 年是 GPU 算力市场的卖方时代。供给严重不足，需求指数级增长，这让早期投入 GPU 基础设施的公司赚到了丰厚的利润。

**A100（Ampere 架构，2020 年发布）**

A100 是第一代大规模用于 AI 训练的数据中心 GPU。OpenAI 用 A100 集群（在 Azure 上）训练了 GPT-3 和 GPT-4。

| 指标 | 数据 |
|------|------|
| 单卡采购成本 | ~$10,000-12,000 |
| 配套部署总成本（含服务器、网络、数据中心分摊） | ~$18,000-22,000/GPU |
| 云租赁峰值价格（2023 年） | ~$3.50-4.00/GPU-hr |
| 70% 利用率下年收入 | ~$21,500-24,500/GPU |
| 回本周期 | **0.8-1.2 年** |
| 折旧后剩余年限的净利润 | 3.8-4.2 年的纯利润 |

A100 的经济模型极其优越：快速回本，然后在接下来的 4 年里几乎全是利润。到 2025 年底，A100 虽已服役 5 年，Azure 仍未宣布退役 A100 VM 系列，仍在以 $1.35/hr 左右的价格出租——虽然价格暴跌，但硬件早已折旧完毕，这个价格几乎全是利润。

**H100（Hopper 架构，2022 年发布，2023 年量产）**

H100 是 AI 训练的分水岭产品。2023 年全年处于极度供不应求的状态，NVIDIA 的数据中心收入从 $15B 暴增至 $47.5B。

| 指标 | 数据 |
|------|------|
| 单卡采购成本 | ~$25,000-30,000 |
| 配套部署总成本 | ~$35,000-45,000/GPU |
| 云租赁峰值价格（2023 年） | ~$8.00+/GPU-hr |
| 70% 利用率下峰值年收入 | ~$49,000/GPU |
| 回本周期（峰值价格） | **0.7-0.9 年** |
| 当前租赁价格（2026.03） | ~$1.90-2.30/GPU-hr（中位数） |
| 当前年收入（70% 利用率） | ~$11,700-14,100/GPU |

H100 是历史上最赚钱的 GPU 之一。2023 年部署的 H100 基本在 1 年内回本，之后的 4 年都在产生利润。即使价格暴跌 70%+，对于早期买家来说无所谓——硬件成本早已收回。

**关键修正：H100 的折旧状态**

H100 大规模出货始于 2023 年 Q3-Q4，按 5 年直线折旧到 2028 年 Q3-Q4。截至 2026 年 3 月，H100 仅完成约 **50% 的折旧**，仍有约 **$0.68/hr** 的 GPU 折旧成本待摊销。这意味着 H100 并不像 A100 那样"全是利润"——当前 $1.90-2.30/hr 的租赁价里，折旧占了约 30-35%。

CoreWeave 在 2023 年用 A100/H100 作为银行贷款抵押品借了 $2.3B，2025 年 IPO 路演时还说 H100 是"sold out"状态——需求最强的产品。

**关键特征：2020-2024 年的 GPU 经济学**

| 特征 | 描述 |
|------|------|
| 供需关系 | 极度短缺，有钱买不到卡 |
| 定价权 | 卖方市场，云厂商定价 |
| 回本速度 | 0.7-1.2 年 |
| 折旧后净利润期 | 3.8-4.3 年（按 5 年折旧） |
| 老卡命运 | 降级为推理卡，仍有价值 |
| 二手市场 | 活跃，价格坚挺 |
| 是否出现"弃旧"现象 | **从未出现** |

在这个时代，投资 GPU 算力基础设施是一笔好生意。A100 和 H100 都大幅跑赢了折旧曲线，为持有者创造了巨额利润。这也是 Oracle、CoreWeave 等公司敢于大举借债买卡的逻辑基础——"历史证明 GPU 一定赚钱"。

### 1.2 A100 的真实运营成本拆解

A100 跌到 $1.35/hr 是否"只够覆盖电费"？让我们精确计算。

**电费计算**

| 项目 | 数值 |
|------|------|
| A100 SXM TDP | 400W |
| PUE（电力使用效率） | 1.3（典型值） |
| 实际消耗功率 | 400W × 1.3 = **520W** |
| 电价 | $0.07/kWh（数据中心批发价） |
| **每小时电费** | 0.52kW × $0.07 = **$0.036** |

$0.036/hr 的纯电费只占 $1.35 租赁价的 **2.7%**。所以"只够覆盖电费"是不准确的说法。

**完整的运营成本构成（A100 已完全折旧后）**

| 成本项 | $/GPU-hr | 占比 | 说明 |
|--------|----------|------|------|
| GPU 折旧 | $0.00 | 0% | 5 年折旧已完成（2020→2025） |
| 服务器折旧（非 GPU） | $0.00 | 0% | 与 GPU 同批采购，同步折旧完成 |
| 电力 | $0.04 | 8-13% | 520W × $0.07/kWh |
| 网络/带宽 | $0.05-0.10 | 15-20% | InfiniBand 交换机折旧 + 带宽 |
| 数据中心租赁/折旧 | $0.10-0.18 | 25-35% | 建筑物按 20-25 年折旧，远未完成 |
| 运维人员 | $0.02-0.05 | 5-10% | 大型 DC 共享运维团队，人均分摊极低 |
| 冷却维护 | $0.03-0.05 | 8-12% | 风冷系统维护 + 设备更换 |
| 保险 + 其他 | $0.02-0.03 | 5-8% | |
| **全运营成本** | **$0.26-0.45** | **100%** | |

> **关键结论**：A100 以 $1.35/hr 出租时，扣除全运营成本后仍有 **$0.90-1.09/hr** 的毛利润（毛利率 67-81%）。Princeton CITP 所说的"keep the lights on"指的是全运营成本（$0.26-0.45），不仅仅是电费（$0.036）。但即便如此，A100 在折旧完成后仍然是盈利的——只是利润率远低于黄金时代。

**运维人员成本修正说明**

最初估算运维人员占比为 4-7%，经核算后下调至 2-5%。原因是大型数据中心（10,000+ GPU）共享一批运维人员：50 人 × $150K 年薪 / 10,000 GPU / 8,760 小时 = **$0.086/hr**；对于更大规模的 DC（如 Abilene 的 45 万张 GPU 规模），人均分摊更低至 $0.02-0.03/hr。

### 1.3 现在不一样了：三个根本性变化同时发生

2025-2026 年，GPU 算力市场的底层逻辑发生了根本转变。三个变化叠加在一起，打破了过去 4 年的黄金定律。

**变化一：NVIDIA 的芯片发布节奏从 2 年加速到 1 年**

| 时期 | 发布节奏 | 架构迭代 |
|------|---------|---------|
| 2020-2023 | **~2 年一代** | Ampere(2020) → Hopper(2022) |
| 2024-2026 | **~1 年一代** | Blackwell(2024) → Vera Rubin(2025) → Rubin Ultra(2026E) |

Jensen Huang 在 CES 2026 上正式确认了"每年一代"的新节奏。这意味着：

- 数据中心的建设周期（12-24 个月）现在**比芯片迭代周期还长**
- 一个数据中心从拿地到通电可能需要 18 个月，但这 18 个月里 NVIDIA 已经发布了 1-2 代新芯片
- **历史上从未出现过这种"硬件比建筑快"的错配**

打个比方：以前盖好一栋房子需要 2 年，家电换代也是 2 年一次，所以房子盖好了家电刚好是最新的。现在家电变成 1 年一换，你房子还没盖好，里面预装的家电就已经过时了。

**变化二：能效差距变得大到让老卡"跑电费都不划算"**

以前的代际升级主要是性能提升（跑得更快），但能效（每瓦性能）的差距不算大，老卡降价后仍有经济性。现在不同了：

| GPU 型号 | 推理性能 (相对值) | 能效 (相对值) | 备注 |
|---------|------------------|-------------|------|
| A100 | 1x | 1x | 基准 |
| H100 | 3-9x | 3-4x | 仍有经济性 |
| GB200 (Blackwell) | 15-30x | 25x | 老卡开始不划算 |
| Vera Rubin | 75-150x | ~100x | **A100/H100 完全不经济** |

**变化三：AI 竞争进入"基准测试排名"时代，客户不再"有卡就行"**

2023 年训练 GPT-4 时，有 A100 就是胜利——什么卡都不嫌。但 2025-2026 年，AI 竞争变成了基准测试排名的军备竞赛，性能上最小的差距都会影响用户量、收入和估值。用更老的芯片训练意味着模型发布时可能已经被竞争对手超越。

**三个变化的叠加效应**

| 时代 | 芯片节奏 | 能效差距 | 客户态度 | 综合效果 |
|------|---------|---------|---------|---------|
| 2020-2024 | 2年/代 | 温和 | 有卡就行 | **GPU 稳赚不赔** |
| 2025-2026 | 1年/代 | 致命 | 只要最新 | **GPU 可能成为搁浅资产** |

这三个变化中的任何一个单独出现，都还可以应对。但三个同时发生，就形成了一个全新的风险：**数据中心还没建好，里面的芯片就已经过时了。**

而 Oracle，恰好是在这个拐点上 all-in 的那个玩家。

### 1.4 能效比深度拆解：从 A100 到 Vera Rubin

#### 1.4.1 B200 vs A100：当代主力对决上一代

B200（Blackwell 架构）是当前已大规模交付并投入使用的最新 GPU 主力。

**绝对性能对比**

| 精度 | A100 | B200 | 倍率 |
|------|------|------|------|
| FP64（双精度） | 9.7 TFLOPS | 40 TFLOPS | 4.1x |
| FP32（单精度） | 19.5 TFLOPS | 80 TFLOPS | 4.1x |
| FP16（半精度） | 312 TFLOPS | 2,250 TFLOPS | 7.2x |
| FP8（推理常用） | 不支持原生 | 4,500 TFLOPS | ∞ |
| FP4（极限推理） | 不支持 | 9,000 TFLOPS | ∞ |

**能效比（TFLOPS/W）对比**

| 精度 | A100 (400W) | B200 (1000W) | B200 能效优势 |
|------|-------------|-------------|-------------|
| FP64 | 0.024 | 0.040 | 1.7x |
| FP32 | 0.049 | 0.080 | 1.6x |
| **FP16** | **0.78** | **2.25** | **2.9x** |
| **FP8** | N/A | **4.50** | **∞（A100 不支持）** |

**实际推理场景的综合能效提升**

在 FP8 推理场景下（当前大模型推理主流精度），B200 相比 A100 的能效提升达到 **10-15 倍**。这意味着完成相同的推理任务，B200 消耗的电力只有 A100 的 **1/10 到 1/15**。

> **关键结论**：B200 对比 A100 的能效比，在 FP16（训练常用精度）下提升 **2.9 倍**，在 FP8 推理下提升 **10-15 倍**。这个差距大到 A100 在电力受限的环境下已经不具备经济性——同样的电力预算，给 B200/GB200 使用的产出是 A100 的 4-15 倍。

#### 1.4.2 Vera Rubin vs GB200：下一代对当代

Vera Rubin（Rubin 架构）是 NVIDIA 的下一代旗舰，预计 2026 年 H2 至 2027 年量产。Oracle 花 $40B 买的 GB200 正面临被它"革命"的风险。

**核心规格对比**

| 指标 | GB200 (Blackwell) | Vera Rubin | 变化 |
|------|-------------------|-----------|------|
| TDP | 1,000W | 2,300W | +130% |
| FP4 算力 | 4,500 TFLOPS | 50,000 TFLOPS (50 PFLOPS) | **11.1x** |
| 显存 | 192GB HBM3e | 288GB HBM4 | +50% |
| 显存带宽 | 8 TB/s | 22 TB/s | 2.75x |
| NVLink 带宽 | 1.8 TB/s | 3.6 TB/s | 2x |

**能效比对比（FP4, TFLOPS/W）**

| GPU | FP4 算力 | 功耗 | 能效（FP4 TFLOPS/W） | 相对 GB200 |
|-----|---------|------|---------------------|-----------|
| GB200 | 4,500 TFLOPS | 1,000W | 4.50 | 1x |
| Vera Rubin | 50,000 TFLOPS | 2,300W | **21.74** | **4.8x** |

NVIDIA 官方宣称的效果更加激进：
- **训练**：Vera Rubin 只需 **1/4 的 GPU 数量** 达到 Blackwell 同等训练效果
- **推理**：每百万 token 的成本降至 Blackwell 的 **1/10**

> **关键结论**：Vera Rubin 对比 GB200 的能效提升为 **4.8 倍**（FP4 每瓦性能）。如果 NVIDIA 的"1/10 推理成本"兑现，这意味着 Oracle 花 $40B 买的 GB200 在 Vera Rubin 面前的竞争力将急剧下降——同样的工作，客户用 Vera Rubin 只需花 1/10 的钱。

### 1.5 Vera Rubin 的安装对比 GB200：表面无缝，实质巨变

#### 1.5.1 机柜层面：设计为"无缝过渡"

NVIDIA 官方产品页明确写道：Vera Rubin NVL72 采用**第三代 MGX NVL72 机架设计**（"Built on the third-generation NVIDIA MGX™ NVL72 rack design, offers a seamless transition from prior generations"）。

- **机柜物理尺寸**：与 GB200 NVL72 相同的 rack form factor
- **计算托盘（compute tray）**：Vera Rubin 采用"无线缆计算托盘设计"（cableless compute tray design），理论上比 GB200 的布线更简洁
- **升级路径**：已装好 GB200 NVL72 机架的 DC，可以"抽屉式"更换 compute tray，不需要换整个机架骨架

所以**纯机械安装层面**，Vera Rubin 在已有 GB200 基础设施上的升级速度可以更快。

#### 1.5.2 但电力和散热是硬约束

| 指标 | GB200 NVL72 | Vera Rubin NVL72 | 变化 |
|------|-------------|------------------|------|
| 单 GPU TDP | 1,000W | 2,300W | **+130%** |
| 整柜功耗 | ~72kW | ~120-130kW | **+67-81%** |
| Power Whip 额定电流 | 60A | 100A | **+67%** |
| 冷却系统流量 | 基准 | ~翻倍（+100%） | **+100%** |
| 单柜冷却成本 | ~$50,000 | ~$56,000+ | **+12-17%** |

这意味着：

- **电力分配单元（PDU）必须升级**：从 60A 升到 100A 的 power whip，需要更换配电铜缆和断路器
- **液冷系统必须大幅扩容**：冷却剂流量翻倍，CDU（Coolant Distribution Unit）的规格和数量都要增加
- **数据中心总电力容量受限**：同样 1.2GW 的 Abilene 园区，如果换成 Vera Rubin，能装的 GPU 总数会**大幅减少**（约少 40-45%）

> **结论**：机柜插拔层面，Vera Rubin 确实设计为"drop-in upgrade"。但数据中心基础设施层面，每个机柜多出 50-60kW 的功耗意味着电力配送和液冷系统必须全面升级，这个改造周期可能需要数月。对 Oracle 来说，把已装好的 GB200 换成 Vera Rubin，不是简单换"芯片"那么简单，整个 DC 的电力和散热骨架都要重做。

### 1.6 历史对比：之前为什么没出现过"弃旧"？

为了理解当前局面的独特性，有必要回顾每一次 GPU 代际过渡中发生了什么。

**A100 → H100（2022-2023）：完美过渡**

- H100 2022年3月发布，但 2023 年才大规模量产
- A100 在 H100 发布后不但没有被嫌弃，二手价格反而**更高了**
- 原因：GPU 极度短缺（"GPU famine"），所有人都在抢卡，新旧通吃
- OpenAI 用 A100 集群训练 GPT-4——不是不想要 H100，是买不到
- **没有任何"放弃 A100"的新闻**

**H100 → Blackwell/GB200（2024-2025）：裂缝初现**

- H100 租赁价格从 >$8/hr 暴跌到 ~$2.10/hr（跌幅 70%+）
- A100 跌到 $1.35/hr
- 但仍然没有客户公开要求"放弃 H100 数据中心"
- CoreWeave 2025 年 IPO 时说 H100"sold out"——因为推理软件栈成熟
- **价格暴跌，但没有弃旧**

**Blackwell → Vera Rubin（2025-2026）：历史首次"弃旧"**

2026年3月，CNBC 报道了一个标志性事件：

> "OpenAI 不再计划扩展与 Oracle 在德克萨斯州 Abilene 的合作。该站点预计使用 NVIDIA 的 Blackwell 处理器，但电力要再过一年才能上线。届时 OpenAI 希望在其他地方使用更大的 NVIDIA 下一代芯片集群。"

这是 **GPU 行业历史上第一次**出现大客户在数据中心建成前公开要求换下一代芯片。

**对比总结**

| | A100→H100 | H100→Blackwell | Blackwell→Rubin |
|---|-----------|----------------|-----------------|
| 供需 | 极度短缺 | 短缺缓解 | 供给追上需求 |
| NVIDIA 节奏 | 2年/代 | 1.5年 | **1年/代** |
| 性能提升 | 3-9x | 2.5-15x | **5x 推理 + 10x 能效** |
| 老卡用途 | 训练+推理都行 | 推理尚可 | **跑电费都不划算** |
| 客户弃旧 | **无** | **无** | **有（OpenAI弃Abilene）** |
| 建设方资金来源 | 经营现金流 | 部分借贷 | **大量举债（Oracle $134B）** |
| 二手市场 | 旺盛 | 走弱 | **可能崩溃** |

### 1.8 是否应该淘汰 A100 换 GB200？

#### 1.8.1 纯经济账：毫无疑问

在电力受限的数据中心里，每一瓦电力都极其宝贵：

| 指标 | A100（400W + PUE = 520W） | GB200（1000W + PUE = 1300W） |
|------|--------------------------|----------------------------|
| FP16 产出 | 312 TFLOPS | 2,500 TFLOPS |
| 每瓦 FP16 产出 | 0.60 TFLOPS/W | 1.92 TFLOPS/W |
| FP8 推理 | 不支持 | 5,000 TFLOPS |
| 每 TFLOP 电费成本 | 基准 | **约 1/5** |

同样的电力预算下，GB200 的产出是 A100 的 **4 倍以上**。在 Abilene 这种电力严重受限的场景（只有 2/8 栋楼运营），每一个机柜位都应该给 GB200 而非 A100。

#### 1.8.2 但现实更复杂

- **A100 已完全折旧**，边际成本只有电费+运维。如果 DC 有多余电力容量，让 A100 继续跑低优先级任务（fine-tuning、数据预处理、非实时推理）仍然合理——硬件成本已经沉没
- **A100 和 GB200 不在同一个物理位置**。A100 在传统风冷 DC，GB200 在 Abilene 的液冷设施。不存在简单的"拔 A100 换 GB200"的操作
- **更现实的做法**：A100 集群继续跑低价值任务（定价越来越低），所有新增投资集中在 GB200/Vera Rubin。A100 会自然"退役"而非被主动拆除

#### 1.8.3 对 Oracle 的含义

如果 A100 的机柜空间和电力资源可以被释放给更高效的 GPU，那确实应该"退役"而非继续运营。这正是 GPU 市场两极分化的微观体现：A100 的残余价值正在趋近于"占用电力的机会成本"。

### 1.9 Oracle 能否退掉未生产完的 GB200？

#### 短答案：几乎不可能。

#### 1.9.1 合同结构锁死

NVIDIA 的大客户 GPU 采购采用 **Non-Cancellable, Non-Refundable（不可撤销、不可退款）** 的订单模式。NVIDIA 在 10-K 财报中明确表述大客户采购承诺为 non-cancellable。

- **预付款**：$40B 的订单，预付比例通常 20-40%，即 $8-16B 已给 NVIDIA，不可退还
- **上游产能已锁定**：NVIDIA 已为 Oracle 锁定 TSMC 的 CoWoS 封装产能和 SK 海力士的 HBM3e 产能，退货意味着全链条违约

#### 1.9.2 退货成本不可承受

| 退货后果 | 估算影响 |
|----------|----------|
| 已付预付款不退 | $8-16B |
| 违约赔偿金 | 合同额 10-20%，即 $4-8B |
| 对 OpenAI 服务违约 | Oracle 承诺提供算力，退 GPU = 违反服务承诺 |
| 信用评级下调 | 已经 BBB-，退货会被解读为"战略失败" |
| 股价冲击 | 市场解读为 AI 战略崩溃 |

#### 1.9.3 Oracle 实际能做的

1. **推迟交付**（最现实）：协商部分批次延后 3-6 个月，NVIDIA 可能同意——可以把产能转卖给 Meta、Google 等急需 GPU 的客户
2. **转售/分配给其他客户**：把部分 GB200 算力卖给 OpenAI 以外的 OCI 云客户，分散风险
3. **协商换代升级**（几乎不可能）：将未交付 GB200 订单转为同等价值的 Vera Rubin，但 Vera Rubin 产能已被 Meta/Microsoft 预定，且单价更高（$8.8M/rack vs ~$3M+/rack），"同等价值"意味着更少 GPU
4. **二手市场出售**（最坏情况）：承认战略失败

> **结论**：Oracle 面对**三重锁定**——合同锁定（$40B 不可退不可换）、基建锁定（Abilene 电力/散热只适配 GB200，换 Vera Rubin 要重做）、客户锁定（OpenAI 是几乎唯一能消化这些算力的大客户，但正要求更新芯片）。$40B 的 GPU 订单已经是板上钉钉的沉没承诺。

**乐观情景**：AI 需求爆发，Oracle OCI 收入推至 $40B+/年，FCF 转正，债务逐步偿还。当前股价被低估。

**悲观情景**：芯片迭代太快导致设施贬值（正在发生），OpenAI 转向其他供应商，AI 需求增速放缓。Oracle 面临设施闲置、债务违约的系统性风险。
