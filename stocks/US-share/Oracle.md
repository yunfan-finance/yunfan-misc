# GPU 算力经济学与 Oracle 财务深度拆解

## 第一部分：GPU 的经济账

### 1.1 过去的黄金时代：A100 和 H100 的盈利现实

2020-2024 年是 GPU 算力市场的卖方时代。供给严重不足，需求指数级增长，这让早期投入 GPU 基础设施的公司赚到了丰厚的利润。

**A100（Ampere 架构，2020 年发布）**

A100 是第一代大规模用于 AI 训练的数据中心 GPU。OpenAI 用 A100 集群（在 Azure 上）训练了 GPT-3 和 GPT-4。

| 指标 | 数据 |
|------|------|
| 单卡采购成本 | ~$10,000-12,000 |
| 配套部署总成本（含服务器、网络、数据中心分摊） | ~$18,000-22,000/GPU |
| 云租赁峰值价格（2023 年） | ~$3.50-4.00/GPU-hr |
| 70% 利用率下年收入 | ~$21,500-24,500/GPU |
| 回本周期 | **0.8-1.2 年** |
| 折旧后剩余年限的净利润 | 3.8-4.2 年的纯利润 |

A100 的经济模型极其优越：快速回本，然后在接下来的 4 年里几乎全是利润。到 2025 年底，A100 虽已服役 5 年，Azure 仍未宣布退役 A100 VM 系列，仍在以 $1.35/hr 左右的价格出租——虽然价格暴跌，但硬件早已折旧完毕，这个价格几乎全是利润。

**H100（Hopper 架构，2022 年发布，2023 年量产）**

H100 是 AI 训练的分水岭产品。2023 年全年处于极度供不应求的状态，NVIDIA 的数据中心收入从 $15B 暴增至 $47.5B。

| 指标 | 数据 |
|------|------|
| 单卡采购成本 | ~$25,000-30,000 |
| 配套部署总成本 | ~$35,000-45,000/GPU |
| 云租赁峰值价格（2023 年） | ~$8.00+/GPU-hr |
| 70% 利用率下峰值年收入 | ~$49,000/GPU |
| 回本周期（峰值价格） | **0.7-0.9 年** |
| 当前租赁价格（2025.12） | ~$2.10/GPU-hr |
| 当前年收入（70% 利用率） | ~$12,900/GPU |

H100 是历史上最赚钱的 GPU 之一。2023 年部署的 H100 基本在 1 年内回本，之后的 4 年都在产生利润。即使 2025 年底价格暴跌 70%，对于早期买家来说无所谓——硬件成本早已收回，$2.10/hr 的价格里大部分仍然是利润。

CoreWeave 在 2023 年用 A100/H100 作为银行贷款抵押品借了 $2.3B，2025 年 IPO 路演时还说 H100 是"sold out"状态——需求最强的产品。

**关键特征：2020-2024 年的 GPU 经济学**

| 特征 | 描述 |
|------|------|
| 供需关系 | 极度短缺，有钱买不到卡 |
| 定价权 | 卖方市场，云厂商定价 |
| 回本速度 | 0.7-1.2 年 |
| 折旧后净利润期 | 3.8-4.3 年（按 5 年折旧） |
| 老卡命运 | 降级为推理卡，仍有价值 |
| 二手市场 | 活跃，价格坚挺 |
| 是否出现"弃旧"现象 | **从未出现** |

在这个时代，投资 GPU 算力基础设施是一笔好生意。A100 和 H100 都大幅跑赢了折旧曲线，为持有者创造了巨额利润。这也是 Oracle、CoreWeave 等公司敢于大举借债买卡的逻辑基础——"历史证明 GPU 一定赚钱"。

### 1.2 现在不一样了：三个根本性变化同时发生

2025-2026 年，GPU 算力市场的底层逻辑发生了根本转变。三个变化叠加在一起，打破了过去 4 年的黄金定律。

**变化一：NVIDIA 的芯片发布节奏从 2 年加速到 1 年**

| 时期 | 发布节奏 | 架构迭代 |
|------|---------|---------|
| 2020-2023 | **~2 年一代** | Ampere(2020) → Hopper(2022) |
| 2024-2026 | **~1 年一代** | Blackwell(2024) → Vera Rubin(2025) → Rubin Ultra(2026E) |

Jensen Huang 在 CES 2026 上正式确认了"每年一代"的新节奏。这意味着：

- 数据中心的建设周期（12-24 个月）现在**比芯片迭代周期还长**
- 一个数据中心从拿地到通电可能需要 18 个月，但这 18 个月里 NVIDIA 已经发布了 1-2 代新芯片
- **历史上从未出现过这种"硬件比建筑快"的错配**

打个比方：以前盖好一栋房子需要 2 年，家电换代也是 2 年一次，所以房子盖好了家电刚好是最新的。现在家电变成 1 年一换，你房子还没盖好，里面预装的家电就已经过时了。

**变化二：能效差距变得大到让老卡"跑电费都不划算"**

以前的代际升级主要是性能提升（跑得更快），但能效（每瓦性能）的差距不算大，老卡降价后仍有经济性。现在不同了：

| GPU 型号 | 推理性能 (相对值) | 能效 (相对值) | 备注 |
|---------|------------------|-------------|------|
| A100 | 1x | 1x | 基准 |
| H100 | 3-9x | 3-4x | 仍有经济性 |
| GB200 (Blackwell) | 15-30x | 25x | 老卡开始不划算 |
| Vera Rubin | 75-150x | ~100x | **A100/H100 完全不经济** |

Princeton CITP 的研究一针见血地指出了问题：

> "以前 GPU 降价只是'新的更好'。现在的能效差距意味着，在电力受限的数据中心里，运行老卡消耗的电费比它能产生的收入还高。这意味着老卡的残值趋近于零，不论折旧表怎么写。"

具体算一笔账：

| 场景 | H100 | GB200 |
|------|------|-------|
| 功耗 | 700W/GPU | 1200W/GPU（但算力 15x+） |
| 每 PFLOPS 功耗 | ~350W | ~40W |
| 电费成本（$0.08/kWh） | $0.49/GPU-hr | $0.84/GPU-hr |
| 出租价格（2025 末） | $2.10/GPU-hr | $6-8/GPU-hr |
| 电费占收入比 | **23%** | **10-14%** |
| 每瓦收入 | $3.00/W | $5-6.7/W |

当 Vera Rubin（每瓦性能再翻数倍）上线后，H100 的每瓦收入将变得**低于电费成本**。届时跑 H100 不仅不赚钱，每跑一小时都在亏钱。理性选择是直接关机。

**变化三：AI 竞争进入"基准测试排名"时代，客户不再"有卡就行"**

2023 年训练 GPT-4 时，有 A100 就是胜利——什么卡都不嫌。那时候的约束是"能不能拿到卡"，而不是"卡够不够新"。

但 2025-2026 年，AI 竞争格局发生了质变：

- 前沿模型的竞争变成了**基准测试排名的军备竞赛**
- 性能上最小的差距都会影响用户量、收入和估值
- OpenAI、Anthropic、Google、xAI 在同一个赛道上按月迭代
- 用更老的芯片训练意味着**模型发布时可能已经被竞争对手超越**

CNBC 引用知情人士的话精确描述了这种逻辑转变：

> "对于训练前沿模型的公司来说，性能上最小的改进都可能转化为模型基准和排名的巨大差距，而这些直接关系到用户量、收入和估值。"

这就是 OpenAI 放弃扩展 Abilene（Blackwell）数据中心的原因——不是 Blackwell 不好，而是等它全面上线时，Vera Rubin 已经量产了。用 Blackwell 训练出来的模型可能在发布前就被用 Rubin 训练的竞品碾压。

**三个变化的叠加效应**

| 时代 | 芯片节奏 | 能效差距 | 客户态度 | 综合效果 |
|------|---------|---------|---------|---------|
| 2020-2024 | 2年/代 | 温和 | 有卡就行 | **GPU 稳赚不赔** |
| 2025-2026 | 1年/代 | 致命 | 只要最新 | **GPU 可能成为搁浅资产** |

这三个变化中的任何一个单独出现，都还可以应对。但三个同时发生，就形成了一个全新的风险：**数据中心还没建好，里面的芯片就已经过时了。** 这是 GPU 算力行业历史上从未出现过的情况。

而 Oracle，恰好是在这个拐点上 all-in 的那个玩家。

### 1.3 历史对比：之前为什么没出现过"弃旧"？

为了理解当前局面的独特性，有必要回顾每一次 GPU 代际过渡中发生了什么。

**A100 → H100（2022-2023）：完美过渡**

- H100 2022年3月发布，但 2023 年才大规模量产
- A100 在 H100 发布后不但没有被嫌弃，二手价格反而**更高了**
- 原因：GPU 极度短缺（"GPU famine"），所有人都在抢卡，新旧通吃
- OpenAI 用 A100 集群训练 GPT-4——不是不想要 H100，是买不到
- CoreWeave 用 A100 做银行贷款抵押品——银行也认为 A100 保值
- **没有任何"放弃 A100"的新闻**

**H100 → Blackwell/GB200（2024-2025）：裂缝初现**

- H100 租赁价格从 >$8/hr 暴跌到 ~$2.10/hr（跌幅 70%+）
- A100 跌到 $1.35/hr——基本只够覆盖电费加微薄利润
- 但仍然没有客户公开要求"放弃 H100 数据中心"
- CoreWeave 2025 年 IPO 时说 H100"sold out"——因为推理软件栈成熟
- H100 虽然不再适合前沿训练，但作为推理卡仍有大量需求
- **价格暴跌，但没有弃旧**

**Blackwell → Vera Rubin（2025-2026）：历史首次"弃旧"**

2026年3月，CNBC 报道了一个标志性事件：

> "OpenAI 不再计划扩展与 Oracle 在德克萨斯州 Abilene 的合作。该站点预计使用 NVIDIA 的 Blackwell 处理器，但电力要再过一年才能上线。届时 OpenAI 希望在其他地方使用更大的 NVIDIA 下一代芯片集群。"

这是 **GPU 行业历史上第一次**出现大客户在数据中心建成前公开要求换下一代芯片。Oracle 已经为 Abilene 投入了数十亿美元的建设和人员成本。

**对比总结**

| | A100→H100 | H100→Blackwell | Blackwell→Rubin |
|---|-----------|----------------|-----------------|
| 供需 | 极度短缺 | 短缺缓解 | 供给追上需求 |
| NVIDIA 节奏 | 2年/代 | 1.5年 | **1年/代** |
| 性能提升 | 3-9x | 2.5-15x | **5x 推理 + 10x 能效** |
| 老卡用途 | 训练+推理都行 | 推理尚可 | **跑电费都不划算** |
| 客户弃旧 | **无** | **无** | **有（OpenAI弃Abilene）** |
| 建设方资金来源 | 经营现金流 | 部分借贷 | **大量举债（Oracle $134B）** |
| 二手市场 | 旺盛 | 走弱 | **可能崩溃** |

Princeton CITP 的研究进一步质疑了 NVIDIA 所说的"老卡可以通过二手市场降级使用"的说法：

> "NVIDIA 的二手市场论点听起来合理——训练卡降级为推理卡，推理卡降级为通用 HPC。但实际数据不支持。H100 和 A100 的租赁价格都已经跌到基本只够覆盖运营成本的水平。硬件实际上在被白送——租户只是在帮卖家交电费。"

## 第二部分：Oracle 的投入拆解

### 2.1 Oracle 的 AI 豪赌全景

Oracle 正在进行一场史无前例的资本扩张。作为传统数据库和 ERP 软件公司，它试图通过大举借债建设数据中心、购买 GPU，在 AI 云市场中争得一席之地。

**核心数据一览（截至 Q3 FY26 / 2026年2月28日）**

| 指标 | 数值 | 同比变化 |
|------|------|---------|
| 总收入（季度） | $17.2B | +22% |
| OCI 云收入（季度） | $4.9B | +84% |
| RPO（剩余履约义务） | $553B | +325% |
| 有息债务 | $134.6B | +$42B（9个月） |
| 表外不可撤销承诺 | ~$262B | +$217B |
| 资本支出（9个月） | $39.2B | +223%（上年同期 $12.1B） |
| 经营现金流（9个月） | $17.4B | +18% |
| **自由现金流（9个月）** | **-$21.8B** | 上年同期 +$2.5B |
| 股价（距2025年9月高点） | 下跌 >50% | — |

Oracle 面临的核心矛盾：**$17.4B 的年化经营现金流要撑起 $50B+ 的年化 Capex**。缺口只能靠借债填补。

---

### 2.2 有息债务拆解：$134.6B 从哪来？

Oracle 的有息债务在 9 个月内从 $92.6B 暴增至 $134.6B，净增 $42B。根据 10-Q 的现金流量表：

**融资活动（FY26 前 9 个月）**

| 融资来源 | 金额 | 说明 |
|---------|------|------|
| 发行高级无抵押票据 + 银团贷款 | $44.5B | 主要融资来源，到期日 2030-2065 年 |
| 发行强制可转换优先股 | $5.0B | 类股权融资，未来转为普通股，稀释股东 |
| 商业票据净增 | $2.3B | 短期周转资金 |
| 偿还到期债务 | -$2.2B | 到期偿还 |
| **净融资额** | **~$49.6B** | |

其中，$44.5B 的债务发行是核心。对比上年同期（$19.5B），增长了 128%。Oracle 在 9 个月内的新增借款金额，超过了很多国家的年 GDP。

**债务结构（估算）**

| 类型 | 估计金额 | 说明 |
|------|---------|------|
| 历史遗留债务（Cerner 收购等） | ~$50-55B | 2022年 $28B 收购 Cerner，加上更早的并购债 |
| AI/OCI 扩张新增债务 | ~$45-50B | FY25-FY26 新发行 |
| 项目融资（银团贷款） | ~$18-20B | 专项用于新墨西哥等 DC 建设 |
| 商业票据 + 其他短期 | ~$10-12B | 周转资金 |
| **合计** | **~$134.6B** | |

**利息负担**

按平均 4-4.5% 的利率估算，年利息支出约 **$5.5-6.0B**。这已经接近 Oracle 一个季度的 OCI 收入。

---

### 2.3 钱花在了哪里？$39.2B Capex 拆解

Oracle 前 9 个月的资本支出为 $39.2B，同比增长 223%。这些钱主要用于：

**资本支出估算分解**

| 用途 | 估计金额 | 说明 |
|------|---------|------|
| **GPU 硬件采购** | **$25-30B** | 向 NVIDIA 购买 ~40 万张 GB200（FT 报道总额 ~$40B，分批交付） |
| 数据中心建设 | $6-10B | Abilene TX（8栋建筑）、新墨西哥、Shackelford County 等 |
| 网络设备 | $2-3B | InfiniBand 交换机、园区互联光纤 |
| 服务器/机架/冷却 | $2-3B | NVL72 机架组装、液冷 CDU、UPS 等 |
| 电力基础设施 | $1-2B | 变电站、备用发电机 |
| **合计** | **~$39.2B** | |

**GPU 是最大的单一支出项**。Oracle 向 NVIDIA 购买了约 40 万张 GB200 GPU，总价约 $40B。这些 GPU 的所有权属于 Oracle，然后以云服务的形式租给 OpenAI 和其他客户。这是 Oracle 债务暴增的根本原因。

### 2.4 表外承诺拆解：$262B 是什么？

除了表内的 $134.6B 有息债务，Oracle 还有 ~$262B 的表外不可撤销承诺（Off-Balance Sheet Unconditional Purchase Obligations），从 FY2025 年的 ~$45B 暴增 480%。

**为什么不在资产负债表上？**

这涉及 US GAAP 的会计处理规则：

| 类别 | 金额 | 会计处理 | 说明 |
|------|------|---------|------|
| **已开始使用的租赁** | $18.5B | **上表**（经营性租赁负债） | 已在运营的数据中心 |
| **未开始的数据中心租约** | ~$150-180B | **不上表** | 数据中心还没建好，"使用权"尚未开始 |
| **电力采购协议 (PPA)** | ~$40-60B | **不上表** | 分类为"采购承诺"而非"租赁" |
| **硬件采购订单** | ~$30-40B | **不上表** | 已下单但未交付的 GPU 等 |
| **网络/冷却/其他** | ~$10-20B | **不上表** | 尚未交付的配套设备 |

通俗来说：你签了一份 5 年后交房的期房合同（总价 500 万），现在只付了 50 万首付。按会计准则，你的负债表上只显示 50 万贷款，但你未来要付的 450 万是"表外承诺"——它是真实的义务，只是还没到记账的时间。

**$262B 的风险等级**

这些承诺是**不可撤销**的（unconditional），Oracle 必须履行。随着数据中心陆续建成投入使用，这 $262B 会逐步转入表内——未来几年 Oracle 的表内负债会持续膨胀。按 10 年平均摊销，年付 ~$26B，已经超过了 Oracle 的年经营现金流（~$22-23B 年化）。

---

### 2.5 Oracle 的硬性金融义务全景

把表内和表外合并来看：

| 类别 | 金额 | 性质 |
|------|------|------|
| 有息债务（表内） | $134.6B | 要还本付息，违约则破产 |
| 经营性租赁（表内） | $18.5B | 已确认的使用权负债 |
| 表外不可撤销承诺 | ~$262B | 必须履行，逐步转入表内 |
| **硬性金融义务合计** | **~$415B** | |

对比 Oracle 的创收能力：

| 指标 | 数值 |
|------|------|
| 年化总收入 | ~$65B |
| 年化经营现金流 | ~$22-23B |
| 年化利息支出 | ~$5.5-6.0B |
| RPO（已签合同待执行） | $553B |

**$415B 的义务 vs $553B 的 RPO**：如果 RPO 能全部按时转化为收入，Oracle 的模型是成立的。但 RPO 是"签了但还没收到钱"，而义务是"签了且必须付钱"。中间存在时间错配和违约风险。

### 2.6 OpenAI 与 Oracle 的协议结构

OpenAI 和 Oracle 之间存在**两层协议**，很多报道混在一起，容易造成混淆。

**第一层：Stargate 合资公司（JV）**

| 要素 | 内容 |
|------|------|
| 宣布时间 | 2025年1月，白宫发布会 |
| 参与方 | OpenAI + SoftBank + Oracle + MGX（阿布扎比主权基金） |
| 总投资承诺 | $500B（4 年） |
| 目标 | 在美国建设 10GW+ AI 数据中心 |
| 性质 | 合资公司，不是简单的买卖合同 |

各方角色：
- **SoftBank**：主要股权投资者，通过 Vision Fund 出资（但 SemiAnalysis 分析认为 SoftBank 实际资金不足）
- **Oracle**：数据中心基础设施建设 + 运营服务提供方，自己承担大量 Capex
- **OpenAI**：算力消费方，贡献技术方向 + 支付使用费
- **MGX**：部分中东站点投资

**第二层：Oracle-OpenAI 直接云合同**

| 要素 | 内容 |
|------|------|
| 报道时间 | 2025年9月10日（WSJ 首发） |
| 性质 | 云服务采购合同（不是 JV 投资） |
| 金额 | ~$300B / 5 年，~$60B/年 |
| 起始时间 | 2027 年开始大规模交付 |
| 内容 | OpenAI 向 Oracle 购买 4.5GW/年的云算力 |

**关键问题：GPU 是谁买的？**

**Oracle 自己买的。** FT 报道 Oracle 花了约 $40B 购买 ~40 万张 GB200。Oracle 拥有这些 GPU 的所有权，然后以云服务形式租给 OpenAI。这是 Oracle 债务暴增的根本原因——它不是"轻资产"地提供平台服务，而是"重资产"地买硬件再转租。

**资金流向图**

```
  SoftBank ──(股权投资)──→ Stargate JV ──(建设资金)──→ 部分数据中心
  Microsoft ──(融资)──→ OpenAI
                           │
                           │ $300B/5yr (云服务费)
                           ▼
                        Oracle ──($40B)──→ NVIDIA (GPU)
                           │
                           ├──($15B+)──→ Crusoe/Lancium (DC建设)
                           │
                           ├──(PPA)──→ 电力公司
                           │
                           └──(利息)──→ 债券持有人
```

**Oracle 所说的"资本轻量化"是什么？**

Oracle CFO 提到的 $29B "capital-light contracts" 只覆盖了**极小部分**合同，主要是指：
- 少数客户自带硬件（BYOH），Oracle 只提供场地和运维
- 部分大客户的预付款

$29B 相比 $553B RPO 只占 **5%**。剩下 95% 仍然是 Oracle 自己出钱建设、自己出钱买 GPU 的传统重资产模式。

### 2.7 Oracle vs 三大云的根本差距

同样是建数据中心，为什么只有 Oracle 靠借债？

| 指标 | Oracle | AWS (Amazon) | Azure (Microsoft) | Google Cloud |
|------|--------|-------------|-------------------|-------------|
| FY26E Capex | ~$50B | ~$105B | ~$80B | ~$75B |
| 年经营现金流 | ~$22B | ~$120B+ | ~$130B+ | ~$110B+ |
| Capex / 经营现金流 | **227%** | 88% | 62% | 68% |
| FCF | **-$28B** | +$15B+ | +$50B+ | +$35B+ |
| Capex 融资来源 | **债务** | 经营现金流 | 经营现金流 | 经营现金流 |
| 信用评级 | Baa2（距垃圾级 2 档） | AA | AA+ | Aa2 |

**核心差距**：AWS/Azure/Google 的数据中心是用赚来的钱建的。Oracle 的数据中心是借来的钱建的。

Amazon 有电商、Microsoft 有 Office/Windows、Google 有搜索广告——这些是年产 $100B+ 现金流的"印钞机"业务，完全可以内部为数据中心扩张提供资金。Oracle 的核心业务（数据库许可 + ERP SaaS）利润率虽高，但现金流规模只有它们的 1/5 到 1/6。

这意味着 Oracle 承受不了任何一项重大风险的兑现——无论是 OpenAI 违约、AI 需求放缓、还是新一代芯片导致现有设施贬值。而三大云即使遇到同样的问题，也有充足的现金流缓冲来吸收损失。

### 2.8 Oracle 的最新风险信号

**1. OpenAI 放弃扩展 Abilene**

CNBC 2026年3月9日报道：OpenAI 已停止扩展 Abilene 数据中心计划。现有站点将使用 Blackwell，但 OpenAI 想要 Vera Rubin，选择在其他地方建设新集群。Oracle 已经为 Abilene 投入了数十亿美元的建设和人员成本。

**2. 融资伙伴退出**

Blue Owl Capital 拒绝为 Oracle 的额外数据中心设施提供融资。当你的融资伙伴开始说"不"的时候，市场信心正在动摇。

**3. 大规模裁员**

Oracle 计划裁员多达 30,000 人，重组费用 $1.6B。一边大举扩张数据中心，一边大规模裁员——这通常是财务压力的信号。

**4. 股价腰斩**

Oracle 股价从 2025 年 9 月高点已下跌超过 50%，是标普 500 表现最差的成分股之一。市场正在重新定价 Oracle 的 AI 故事。

**5. 信用评级压力**

Moody's 警告 Oracle 的杠杆率可能升至 4x EBITDA，并给出负面展望。如果被降级至 Ba1（垃圾级），Oracle 的再融资成本将大幅上升，形成恶性循环。

### 2.9 Oracle 的赌局结构

Oracle 当前的处境可以用一句话总结：**用 $134.6B 的表内债务 + $262B 的表外承诺，赌 AI 算力需求在未来 5 年持续高速增长。**

| 赌局要素 | 内容 |
|---------|------|
| **投入** | $134.6B 有息债务 + $262B 表外承诺 = ~$397B 硬性义务 |
| **押注** | $553B RPO 能按时转化为收入 |
| **赢的条件** | AI 需求持续爆发 → 年收入 $80-100B → 债务可控 → FCF 转正 |
| **输的条件** | AI 放缓 / OpenAI 违约 / 芯片过时 → 设施闲置 → 利息吞噬利润 → 信用降级 |
| **时间窗口** | FY2028（2027年中）必须实现 FCF 转正，否则再融资压力剧增 |
| **生死线** | 如果到 FY2029 仍然 FCF 为负，债务滚动将难以为继 |

**乐观情景**：AI 需求如预期爆发，OpenAI + Meta + 其他客户的使用量将 Oracle OCI 收入推至 $40B+/年，FCF 在 FY2028 转正，债务逐步偿还。在这个情景下，Oracle 当前的股价是被低估的。

**悲观情景**：芯片迭代太快导致现有设施贬值（正在发生），OpenAI 转向其他供应商或自建基础设施，AI 需求增速放缓。在这个情景下，Oracle 将面临设施闲置、债务违约的系统性风险。$134.6B 的债务在 Baa2 评级下仍可管理，但一旦降级至垃圾级，融资成本飙升将形成死亡螺旋。
